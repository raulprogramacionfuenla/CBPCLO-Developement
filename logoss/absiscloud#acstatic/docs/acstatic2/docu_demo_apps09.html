Listados informativos

Listado de noticias

Cabecera que generará el contenedor

El listado informativo permite mostrar anuncios breves, con un enlace.

Para definis un listado informativo, definimos un componente div que tenga como class el valor news-list

Dicho componente será una lista de noticias, representadas en forma de bloques div con la clase news-item

El contenido de este bloque dependerá de si queremos que la noticia tenga una imagen asociada o tan solo un texto

  • Noticia sin imágenes: Definiremos los siguientes 3 componentes
    • Un bloque de tipo a con la clase news-title. Tiene las siguientes peculiaridades:
      • El atributo href debe contener el enlace de la noticia
      • El cuerpo del bloque debe contener el texto que queremos visualizar como título (en azul)
    • Un bloque de tipo p con la clase news-subtitle que contiene el subtítulo.
    • Un bloque de tipo p con la clase news-resume con el cuerpo de la noticia.
  • Noticia con imágenes: Definiremos estos 2 componentes
    • Un bloque de tipo div con la clase news-image-container con los siguientes elementos
      • Un componente img con class news-image donde el atributo src apunte a la imagen que queremos mostrar
      • Un bloque p con class news-image-footer que contenga la nota a pie de imagen.
    • Un bloque de tipo div con la clase news-content-container. Este bloque debe contener exactamente los mismos tres componentes que contendría la noticia si no tuviera imágenes.
      • Un bloque de tipo a con la clase news-title. Tiene las siguientes peculiaridades:
        • El atributo href debe contener el enlace de la noticia
        • El cuerpo del bloque debe contener el texto que queremos visualizar como título (en azul)
      • Un bloque de tipo p con la clase news-subtitle que contiene el subtítulo.
      • Un bloque de tipo p con la clase news-resume con el cuerpo de la noticia.
											
<div class="news-list">
	<div class="news-item">
		<a class="news-title" href="#">Las nuevas pantallas de 22" contendrán Caixa Apps</a>
		<p class="news-subtitle">Durante el proceso de migración de windows 7 y sustitución de pantallas 22" panorámicas.</p>
		<p class="news-resume">También se añadirá en el nuevo espacio disponible pequeñas aplicaciones que servirán para hacer más accesible las herramientas del Terminal.</p>
	</div>
	<div class="news-item">
		<div class="news-image-container">
			<img class="news-image" src="img/pantalla.jpg"/>
			<p class="news-image-footer">Sony EX308 720p LCD HDTV</p>
		</div>
		<div class="news-content-container">
			<a class="news-title" href="#">Las nuevas pantallas de 22" contendrán Caixa Apps</a>
			<p class="news-subtitle">Durante el proceso de migración de windows 7 y sustitución de pantallas 22" panorámicas.</p>
			<p class="news-resume">También se añadirá en el nuevo espacio disponible pequeñas aplicaciones que servirán para hacer más accesible las herramientas del Terminal.</p>
		</div>
	</div>
</div>

Listado de incidencias

Cabecera que generará el contenedor

El listado de incidencias nos permite visualizar de forma estructurada un conjunto de incidencias, tanto resueltas como pendientes de resolver.

Para definir distintos listados de incidencias (uno para las cerradas y otro para las pendientes, como en el ejemplo) podemos usar varios bloques de clase widget-control-group. Para incrustar un icono al lado del título, consultar el muestrario de iconos

Dentro de cada bloque, definimos un componente div que tenga como class el valor issue-list

Dicho componente será una lista de incidencias, representadas en forma de bloques div con la clase issue-entry

Cada una de estas incidencias va a contener 2 elementos:

  • Cabecera: Consiste en un bloque de tipo div con la clase issue-entry-header que contiene lo siguiente:
    • Un bloque de tipo a con la clase issue-entry-title. Tiene las siguientes peculiaridades:
      • El atributo href debe contener el enlace de la incidencia
      • El cuerpo del bloque debe contener el texto que queremos visualizar como título (en azul)
    • Un bloque de tipo span con la clase issue-entry-date que contiene la fecha. Este campo es opcional.
  • Cuerpo: Consiste en un bloque de tipo div con la clase issue-entry-abstract que contiene el resumen de la incidencia: 
											
<div class="widget-control-group">
	<div class="widget-control-group-title">
		<i class="icono-correcto"></i>Incidencias cerradas
	</div>
	<div class="widget-control-group-content">
		<div class="issue-list">
			<div class="issue-entry">
				<div class="issue-entry-header">
					<a class="issue-entry-title"
						href="#">
						Contrato financiero atípico extructurado mixto 90/10 no disponible</a>
					<span class="issue-entry-date">- 15-04-2010 12:04h</span>
				</div>
				<div class="issue-entry-abstract">
					Disponible a partir de mañana. Los directores pueden solicitar las autorizaciones al delegado general.
				</div>
			</div>
		</div>
	</div>
</div>
