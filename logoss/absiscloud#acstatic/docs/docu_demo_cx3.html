El uso de las tablas es idéntico al del punto anterior (Datatables Row Grouping), la diferencia es que en este caso de uso se muestra como personalizar las cabeceras.

ABSIS Cloud proporciona un mecanismo que inserta automáticamente una tabla por cabecera de agrupación. El mecanismo consiste en crear una tabla con las cabeceras que queremos insertar. Esta tabla deberá contener varios atributos de tipo data-* que le ofrecerán al sistema la información necesaria para hacer la fusión de la tabla con las cabeceras.

El tag table tabla deberá contener un atributo con data-role="grouping-headers" y otro llamado data-table-id que tenga por valor el identificador de la tabla, por ejemplo, si queremos inyectar la cabecera a la tabla idTabla tendríamos: data-table-id="idTabla".

Cada tag TR de la tabla de cabeceras deberá contener la información que se quiera meter dentro de los TR de agrupación. La substitución se realizará cuando se encuentren las ocurrencias del contenido data-locator-text del TR de cabeceras en el TD la agrupación. Por ejemplo, si vamos a realizar una agrupación por el texto Realizada, el data-locator-text debería ser "Realizada" (data-locator-text="Realizada"). 


<table data-role="grouping-headers" data-table-id="id-tabla">
	<tr  data-locator-text="Realizada">
		<td><a href="#123">Realizada</a></td>
		<td></td>
		<td>2</td>
		<td>3</td>
	</tr>
	...
</table>

Finalmente añadir que en la configuración del rowGrouping será necesario rellenar el atributo fnOnGrouped con una función que termine invocando a la funcion JS applyRowGroupingHeaders();. Véase el siguiente ejemplo: 


 
/* Inicialización tabla */
$(document).ready(function() {
	$(document).ready( function () {
		$('#id-tabla').dataTable({ 
				"bLengthChange": false, 
				"bPaginate": false,
				"bFilter": false
		}).rowGrouping({
				bExpandableGrouping: true, 
				asExpandedGroups: ["Planificada"],
				fnOnGrouped: function(){
					applyRowGroupingHeaders(); 
				}
			});
	} );
</script>
