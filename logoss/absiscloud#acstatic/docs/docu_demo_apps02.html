Elementos básicos

Subtítulo de contenido

Cabecera que generará el contenedor

En la página Estructura del widget se explica el lay-out que establece las dimensiones del widget. El contenido específico de nuestra página debe entrarse dentro del DIV de clase widget-body.

En dicho DIV crearemos uno o más DIVs de clase widget-control-group, cada uno de los cuales destinado a mostrar un contenido, componente o control concreto dentro de un DIV de clase widget-control-group-content.

Opcionalmente puede mostrarse por encima el subtítulo específico de dicho contenido indicándolo en un DIV de clase widget-control-group-title. 


<div class="widget-inner widget-inner-hpattern-4-4 widget-inner-vpattern-2-4">
	<div class="widget-header">
	</div>
	<div class="widget-body widget-body-vpattern-2-4">

		<div class="widget-control-group">
			<div class="widget-control-group-title">
				Subtítulo de contenido
			</div>
			<div class="widget-control-group-content">
				<p>Lorem ipsum dolor sit amet...
			</div>
		</div>

	</div>
	<div class="widget-footer">
	</div>
</div>

Enlaces

Cabecera que generará el contenedor

Para mostrar un enlace en cualquier lugar simplemente haremos uso del tag estándar <a> de HTML: 


<div class="widget-control-group-content">
	<p><a href="#">Ejemplo de enlace</a></p>
</div>

Para mostrar un lista de enlaces, usaremos el tag estándar <a> en una lista estándar HTML: 


<div class="widget-control-group-content">
	<ul>
		<li><a href="#">Elemento enlazable 1</a></li>
		<li><a href="#">Elemento enlazable 2</a></li>
		<li><a href="#">Elemento enlazable n</a></li>
	</ul>
</div>

Para mostrar un enlace con una viñeta, a la izquierda del mismo haremos aparecer un icono de clase icono-bullet-fletxa-gran: 


<div class="widget-control-group-content">
	<p><i class="icono-bullet-fletxa-gran"></i> <a href="#">Ejemplo de enlace navegacional</a></p>
</div>

Botones

Cabecera que generará el contenedor

Para mostrar un botón usaremos el tag <button> de tipo submit indicando clase btnAPPS: 


<div class="widget-control-group-content">
	<button class="btnAPPS" type="submit">Guardar</button>
</div>

Texto mono-linea

Cabecera que generará el contenedor

Para mostrar un campo de entrada usaremos el tag <input> de tipo text. Si queremos mostrar un texto indicativo o de ayuda, informaremos un atributo default en el tag: 


<div class="widget-control-group-content">
	<label for="input01" class="control-label">Texto:</label>
	<input type="text" id="input01" default="Escriba una nueva nota..." />
</div>

Opcionalmente puede mostrarse a la izquierda del campo de entrada una etiqueta. Para ello usaremos el tag label indicando clase control-label.

Adicionalmente podemos añadir las clases css error o disable al div contenedor si queremos aplicar los estilos de error o deshabilitado. 


<div class="widget-control-group-content error">
	<label for="input01" class="control-label">Texto:</label>
	<input type="text" id="input01" value="valor" />
</div>

Para que se aplique el comportamiento dinámico será necesario incorporar al cerrar el body de la página los ficheros javascript siguientes: 


</body>

Buscador

Cabecera que generará el contenedor

Para mostrar un buscador, definiremos un bloque form en el cual especificaremos los siguientes atributos:

  • class: debe valer search-form
  • method: debe valer GET
  • action: debe contener la URL donde se procesará la búsqueda

Además de definir estos atributos, el bloque contendrá los siguientes elementos:

  • Un bloque de tipo input con los siguientes atributos:
    • type debe tener el valor text
    • id debe tener un valor único, ya que se utilizará como identificador
    • name debe tener exactamente el mismo valor que el campo id
    • default debe tener un valor de texto, que se mostrará cuando no haya texto escrito en el campo de búsqueda
  • Un bloque de tipo button con los siguientes atributos:
    • type debe tener el valor submit
    • class debe tener el valor btnAPPS
    • title debe tener un valor de texto, que se mostrará en el botón de búsqueda
    • El cuerpo del componente debe tener el mismo valor que el campo title

<form class="search-form" method="GET" action="#">
	<input type="text" id="input01" name="input01" default="Numero de norma o texto a buscar..." />
	<button title="Buscar" class="btnAPPS" type="submit">Buscar</button>
</form>

Para que se aplique el comportamiento dinámico será necesario incorporar al cerrar el body de la página los ficheros javascript siguientes: 


</body>
